## 二、第一批重构：修入口、修数据分支、修 Trainer bug

### 1. 重构 `train.py`
文件：
- `algorithm/FoundIR/train.py`

主要改动：
- 去掉 `sys.argv[1]` 读取 `sampling_timesteps` 的写法。
- 去掉脚本内硬编码的 `CUDA_VISIBLE_DEVICES`。
- 将入口改为正式 `argparse` 风格。
- 新增显式参数：
  - `--dataroot`
  - `--meta`
  - `--dataset_mode`
  - `--image_size`
  - `--sampling_timesteps`
  - `--timesteps`
  - `--train_num_steps`
  - `--results_folder`
  - `--resume_milestone`
  - 以及训练相关超参数
- 新增 `main()` 和 `build_dataset()`，使脚本结构更清晰。

本次收益：
- 训练入口不再依赖硬编码环境。
- 参数来源统一。
- 后续可直接用命令行控制训练实验，而不是改源码。

### 2. 重构 `test.py`
文件：
- `algorithm/FoundIR/test.py`

主要改动：
- 去掉旧版“for our testset / for your own data”手动注释切换逻辑。
- 改成正式 `argparse` 入口。
- 新增显式参数：
  - `--dataroot`
  - `--meta`
  - `--dataset_mode`
  - `--checkpoint_dir`
  - `--checkpoint_milestone`
  - `--output_dir`
  - `--crop_phase`
  - `--crop_stride`
  - `--sampling_timesteps`
  - `--image_size`
- 新增 `main()` 和 `build_dataset()`。
- 将 checkpoint 加载从写死的 `trainer.load(2000)` 改为显式参数控制。

本次收益：
- 测试入口从“实验脚本”提升为“可复用推理入口”。
- 不再依赖人工改代码切换测试场景。

### 3. 清理 `CombinedDataset` 初始化逻辑
文件：
- `algorithm/FoundIR/data/combined_dataset.py`

主要改动：
- 将 `task` 处理改为显式模式：
  - `paired`
  - `meta_info`
- 去掉原先容易误解的双 `if` + `else` 结构。
- 当 `task='meta_info'` 且未传 `opt.meta` 时显式报错。
- 统一在初始化后设置：
  - `self.A_size`
  - `self.B_size`
- `__getitem__` 中按 `self.task` 直接分流，不再附加冗余条件。

本次收益：
- 数据分支更明确。
- 后续扩展自己的数据路径和 meta 格式时更容易控制行为。

### 4. 修复 `Trainer` 中的条件分支 bug
文件：
- `algorithm/FoundIR/src/model.py`

主要改动：
- 为 `Trainer.__init__()` 增加参数：
  - `data_mode`
- 将原来依赖 `results_folder` 字符串判断数据模式的逻辑改为显式 `data_mode`。
- 修复原先错误代码：
```python
if 'combined' or 'all' in results_folder:
```
- 当前改为：
  - `self.data_mode in ('combined', 'all', 'paired', 'meta_info')`
  - 或 `self.data_mode == 'paired_split'`

本次收益：
- 修掉了真实行为 bug。
- 不再靠输出目录名决定训练流程。
- 配置语义和路径命名解耦。

### 5. 语法检查
已通过：
```bash
python -m py_compile \
  algorithm/FoundIR/train.py \
  algorithm/FoundIR/test.py \
  algorithm/FoundIR/data/combined_dataset.py \
  algorithm/FoundIR/src/model.py
```

## 七、第三批重构（已执行）

### 1. 将 `Trainer` 从 `src/model.py` 中拆出

修改文件：
- `algorithm/FoundIR/train.py`
- `algorithm/FoundIR/test.py`

主要改动：
- 原先从 `src.model` 导入 `Trainer`
- 现在改为：
  - `from src.trainer import Trainer`

本次收益：
- 入口脚本职责更清晰。
- 后续继续整理 `Trainer` 时，不需要再反复动扩散主体文件。

### 3. 收缩 `src/model.py`
修改文件：
- `algorithm/FoundIR/src/model.py`

主要改动：
- 删除原地定义的 `Trainer` 类。
- 使该文件更接近：
  - 核心网络结构
  - 扩散过程实现
  - 通用辅助函数

本次收益：
- `model.py` 的职责边界更合理。

### 4. 新模块依赖关系
当前关系：
- `train.py` / `test.py` 依赖：
  - `src.model`
  - `src.trainer`

- 当前仍存在“Trainer 对 model 内工具函数的依赖”，但这已经比原先全部挤在一个文件里更清晰。
