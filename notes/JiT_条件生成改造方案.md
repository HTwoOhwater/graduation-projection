# JiT 改造成条件生成重建模型的方案（基于 JiT 原生架构硬核定制）

## 1. 目标与边界

tmux  JiT 从“类别条件的扩散生成模型”改造成“图像条件的重建模型”（如去雾任务），**同时严格保留 JiT 论文的所有核心创新机制不变。**

**绝不妥协、不魔改 JiT 的以下核心特征**：
1. **纯 ViT 像素级架构**：不加 U-Net 跳层，不搞 VAE 降维。
2. **训练目标与物理意义**：不改去预测“残差（pred_res）”或“图像本身（pred_x0）”，**坚决坚持 JiT 的原生动力学创新——预测速度场 $v$**。
3. **损失函数边界（可调但不改目标）**：训练目标始终是速度场 $v$，可在 L2 与 L1（或二者加权）之间切换；不添加 SSIM、感知等多重辅助损失。
4. **条件注入灵魂**：推崇原生 In-Context Token 序列拼接式融合，不加冗余的 Cross-Attention。

tmux
- 输入：带噪状态 $z_t$ 与 条件退化图 $y_{cond}$。
- 目标：拟合扩散 ODE 的速度场 $v$。
- 输出：利用微分方程求解器（如 Euler/Heun）结合条件图顺移轨道完成重建。

## 2. 总体改造路线

### 阶段 A：最小可跑条件化（验证数据与连通性）
 -> 得到网络输出 -> 更新模型”的闭环。

2. 在进入模型最前端，采用“通道拼接”（$cat([z_t, y_{cond}], dim=1)$，变成 6 通道）。
3. 训练过程的目标保持不变（即 $v\_pred$ 对齐真实 $v$）；惩罚形式允许在 L2/L1/加权 L1+L2 间切换。

### 阶段 B：采样与重构推理的“条件化定向”（引入图像恢复策略）
 **推理采样端（Sampling）** 下功夫，将它从随机生成引擎变为精确定向的化雾器。
tmux
1. **采样起点改造（条件邻域初始化）**：不从纯高斯噪声 $z_T \sim \mathcal{N}(0, I)$ 起手！而是利用退化图像引入微弱定向信息：即 $z_T = \alpha \cdot y_{cond} + \sigma \cdot noise$（类似 SDEdit 的原理），从而缩减无用的探索路径。
2. **采样步数精简**：测试使用 10 到 20 步的高效求解器（保留 `_euler_step` 和 `_heun_step`），因为有强大的输入前置条件，ODE 的路径将远比无条件生成更加平滑直线。

### 阶段 C：完全解锁 JiT 原生条件注入（In-Context Token 级融合）
tmux     A 的通道拼接 6 通道妥协做法，在条件引入上也要实现最纯正的 Transformer 信仰。
tmux
1. 将退化图 $y_{cond}$ 同样使用 Patch Embedding 转成一系列 Token `cond_tokens`。
2. 在核心代码中，将 `cond_tokens` 像大语言模型的上下文提示词（Prompt）一样，直接在前段与噪声特征 `x_tokens` 拼接：`tokens = torch.cat([cond_tokens, x_tokens], dim=1)`。
3. 让 Transformer 的 Self-Attention 层自我完成图像和噪声图的空间依赖映射关系。
4. 预测与反向传播依旧雷打不动地预测属于图像 Token 部分的速度 $v$。
